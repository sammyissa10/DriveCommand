  <name>Task 1: Create reusable DocumentUpload and DocumentList client components</name>
  <files>
    src/components/documents/document-upload.tsx

  <action>
    1. Create `src/components/documents/document-upload.tsx`:
       - `'use client'` component.
       - Props: `entityType: 'truck' | 'route'`, `entityId: string`, `onUploadComplete: () => void` (callback to refresh list).
       - State: `uploading` (boolean), `error` (string | null), `progress` ('idle' | 'validating' | 'uploading' | 'saving').
       - Render a file input with accept=".pdf,.jpg,.jpeg,.png" and a styled label/button.
       - On file selection:
         a. Set progress to 'validating'.
         b. Client-side pre-validation: check file.size <= 10MB, check file.type is in allowed list. Show error if invalid.
         c. Create FormData with file, entityType, entityId.
         d. Call `requestUploadUrl(formData)` server action.
         e. If error returned, display it and reset.
         f. Set progress to 'uploading'.
         g. Upload file directly to S3 via `fetch(uploadUrl, { method: 'PUT', body: file, headers: { 'Content-Type': contentType } })`.
         h. If upload fails, display error and reset.
         i. Set progress to 'saving'.
         j. Call `completeUpload({ s3Key, fileName, contentType, sizeBytes, entityType, entityId })`.
         k. If success, reset state and call onUploadComplete().
       - Show progress indicator: "Validating..." / "Uploading..." / "Saving..." based on progress state.
       - Show file type and size constraints as helper text below the input: "PDF, JPEG, or PNG. Max 10MB."
       - Style with Tailwind: border-dashed upload area, blue accent consistent with existing UI.

    2. Create `src/components/documents/document-list.tsx`:
       - `'use client'` component.
       - Props: `documents: Array<{ id, fileName, contentType, sizeBytes, createdAt }>`, `onDocumentDeleted: () => void`.
       - Use `useOptimistic` for delete operations (follow existing pattern from truck list).
       - For each document, render a row with:
         - File name (truncated if long)
         - File type icon or badge (PDF icon, Image icon -- simple text badges are fine: "PDF", "JPEG", "PNG")
         - File size formatted (KB/MB)
         - Upload date formatted
         - Download button: calls `getDownloadUrl(docId)`, then opens URL in new tab via `window.open(downloadUrl, '_blank')`
         - Delete button: confirmation prompt, then calls `deleteDocument(docId)`, optimistic removal from list, calls onDocumentDeleted()
       - Empty state: "No documents uploaded yet." message.
       - Pending items (optimistic) shown with reduced opacity (opacity-50 pattern from existing code).
       - Format file size: bytes -> "1.2 MB" or "450 KB".
       - Style with Tailwind matching existing card/list patterns (white bg, border, rounded).
  </action>
