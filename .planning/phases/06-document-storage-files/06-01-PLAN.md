  <action>
    1. Install dependencies:
       `npm install @aws-sdk/client-s3 @aws-sdk/s3-request-presigner file-type nanoid`

    2. Update `next.config.ts` to set `serverActions.bodySizeLimit` to `'10mb'` -- needed even with presigned URLs because the initial FormData with the file slice (4100 bytes for magic bytes) goes through server actions.

    3. Create `src/lib/storage/s3-client.ts`:
       - Export a singleton S3Client configured from env vars: S3_ENDPOINT (optional, for R2), S3_REGION, S3_ACCESS_KEY_ID, S3_SECRET_ACCESS_KEY.
       - If S3_ENDPOINT is set, use it (Cloudflare R2). If not, default AWS behavior.
       - Use `forcePathStyle: true` when endpoint is set (required for R2).
       - Export the bucket name from S3_BUCKET env var.

    4. Create `src/lib/storage/validate.ts`:
       - Export `ALLOWED_TYPES` map: `{ 'application/pdf': ['.pdf'], 'image/jpeg': ['.jpg', '.jpeg'], 'image/png': ['.png'] }`.
       - Export `MAX_FILE_SIZE = 10 * 1024 * 1024` (10MB).
       - Export `async function validateFileType(buffer: ArrayBuffer, claimedType: string)`: Uses `fileTypeFromBuffer` from `file-type` to detect actual type from magic bytes. Returns `{ valid: true, detectedType: string }` or `{ valid: false, error: string }`. Rejects if: magic bytes undetectable, detected type not in ALLOWED_TYPES, or detected type doesn't match claimed type.
       - Note: file-type is ESM-only. Use dynamic import `const { fileTypeFromBuffer } = await import('file-type')` to avoid CJS issues in Next.js server actions.

    5. Create `src/lib/storage/presigned.ts`:
       - Export `async function generateUploadUrl(tenantId: string, category: 'trucks' | 'routes', fileId: string, fileName: string, contentType: string, fileSize: number)`: Creates PutObjectCommand with tenant-prefixed key `tenant-${tenantId}/${category}/${fileId}-${fileName}`, returns `{ uploadUrl, s3Key }` with 5-minute expiry.
       - Export `async function generateDownloadUrl(s3Key: string)`: Creates GetObjectCommand, returns presigned URL with 1-hour expiry.
       - Export `async function deleteS3Object(s3Key: string)`: Creates DeleteObjectCommand, deletes the object from S3.
       - All functions import s3Client and bucket from s3-client.ts.

    6. Create `src/lib/validations/document.schemas.ts`:
       - Import zod.
       - Export `documentCreateSchema` validating: fileName (string, min 1, max 255), s3Key (string, min 1), contentType (string, must be one of allowed types), sizeBytes (number, positive, max 10MB), truckId (uuid, optional), routeId (uuid, optional). Add a refine: exactly one of truckId or routeId must be provided.
       - Export `DocumentCategory` type: `'trucks' | 'routes'`.

    7. Add Document model to `prisma/schema.prisma`:
       ```
       model Document {
         id          String   @id @default(dbgenerated("gen_random_uuid()")) @db.Uuid
         tenantId    String   @db.Uuid
         truckId     String?  @db.Uuid
         routeId     String?  @db.Uuid
         fileName    String
         s3Key       String
         contentType String
         sizeBytes   Int
         uploadedBy  String   @db.Uuid
         createdAt   DateTime @default(now()) @db.Timestamptz
         updatedAt   DateTime @updatedAt @db.Timestamptz

         tenant Tenant @relation(fields: [tenantId], references: [id])
         truck  Truck?  @relation(fields: [truckId], references: [id])
         route  Route?  @relation(fields: [routeId], references: [id])
         uploader User  @relation(fields: [uploadedBy], references: [id])

         @@index([tenantId])
         @@index([truckId])
         @@index([routeId])
         @@index([uploadedBy])
       }
       ```
       Add `documents Document[]` relation arrays to Tenant, Truck, Route, and User models. For User, the relation field name should be `uploadedDocuments` to avoid conflict with `assignedRoutes`.

    8. Create migration manually at `prisma/migrations/20260214000004_add_document_model/migration.sql`:
       - CREATE TABLE "Document" with all columns matching the schema.
       - CREATE INDEX statements for tenantId, truckId, routeId, uploadedBy.
       - ADD FOREIGN KEY constraints for all relations.
       - Enable RLS: `ALTER TABLE "Document" ENABLE ROW LEVEL SECURITY; ALTER TABLE "Document" FORCE ROW LEVEL SECURITY;`
       - Create tenant isolation policy (same pattern as Route migration): `CREATE POLICY tenant_isolation_policy ON "Document" FOR ALL USING ("tenantId" = current_tenant_id()) WITH CHECK ("tenantId" = current_tenant_id());`
       - Create bypass policy: `CREATE POLICY bypass_rls_policy ON "Document" FOR ALL USING (current_setting('app.bypass_rls', TRUE)::text = 'on');`

    9. Run `npx prisma generate` to regenerate the Prisma client with the Document model.
  </action>

<task type="auto">
  <name>Task 2: Document repository and server actions for upload, download, delete, and list</name>
  <files>

  <action>
    1. Create `src/lib/db/repositories/document.repository.ts`:
       - Extend TenantRepository (same pattern as TruckRepository).
       - Methods:
         - `findByTruckId(truckId: string)`: Returns all documents for a truck, ordered by createdAt desc.
         - `findByRouteId(routeId: string)`: Returns all documents for a route, ordered by createdAt desc.
         - `findById(id: string)`: Returns single document or null.
         - `create(data: { tenantId, truckId?, routeId?, fileName, s3Key, contentType, sizeBytes, uploadedBy })`: Creates document record.
         - `delete(id: string)`: Deletes document record. Returns the deleted record (so caller can get s3Key for S3 cleanup).
       - Use `@ts-ignore` for extended Prisma client calls (same pattern as TruckRepository).

    2. Create `src/app/(owner)/actions/documents.ts`:
       - `'use server'` directive at top.

       - `requestUploadUrl(formData: FormData)`:
         - requireRole([OWNER, MANAGER])
         - Extract: file (as File), entityType ('truck' | 'route'), entityId (string)
         - Validate file size against MAX_FILE_SIZE
         - Read first 4100 bytes: `const buffer = await file.slice(0, 4100).arrayBuffer()`
         - Call validateFileType(buffer, file.type) -- reject if invalid
         - Get tenantId from requireTenantId()
         - Generate fileId with nanoid()
         - Determine category from entityType: 'trucks' or 'routes'
         - Call generateUploadUrl(tenantId, category, fileId, sanitized filename, detectedType, file.size)
         - Return { uploadUrl, s3Key, fileId, fileName: file.name, contentType, sizeBytes: file.size, entityType, entityId }

       - `completeUpload(data: { s3Key, fileName, contentType, sizeBytes, entityType, entityId })`:
         - requireRole([OWNER, MANAGER])
         - Get tenantId from requireTenantId()
         - CRITICAL: Verify s3Key starts with `tenant-${tenantId}/` (defense in depth against path manipulation)
         - Get current user via getCurrentUser() for uploadedBy
         - Validate with documentCreateSchema
         - Create DocumentRepository(tenantId)
         - repo.create({ tenantId, fileName, s3Key, contentType, sizeBytes, uploadedBy: user.id, truckId or routeId based on entityType })
         - revalidatePath for the entity detail page (/trucks/[id] or /routes/[id])
         - Return { success: true, document: created record }

       - `getDownloadUrl(documentId: string)`:
         - requireRole([OWNER, MANAGER, DRIVER]) -- drivers need to view route documents
         - Get tenantId from requireTenantId()
         - Create DocumentRepository(tenantId)
         - repo.findById(documentId) -- returns null if not found or wrong tenant (RLS)
         - CRITICAL: Verify doc.s3Key starts with `tenant-${tenantId}/` (defense in depth)
         - Call generateDownloadUrl(doc.s3Key)
         - Return { downloadUrl, fileName: doc.fileName }

       - `deleteDocument(documentId: string)`:
         - requireRole([OWNER, MANAGER])
         - Get tenantId from requireTenantId()
         - Create DocumentRepository(tenantId)
         - Find document first to get s3Key
         - CRITICAL: Verify doc.s3Key starts with `tenant-${tenantId}/`
         - Delete from S3: deleteS3Object(doc.s3Key)
         - Delete from database: repo.delete(documentId)
         - revalidatePath for entity detail page
         - Return { success: true }

       - `listDocuments(entityType: 'truck' | 'route', entityId: string)`:
         - requireRole([OWNER, MANAGER, DRIVER])
         - Get tenantId from requireTenantId()
         - Create DocumentRepository(tenantId)
         - Call repo.findByTruckId or repo.findByRouteId based on entityType
         - Return documents array

    All server actions follow the established pattern: auth check FIRST, then tenant context, then business logic. Use the same error handling style as trucks.ts and routes.ts (return { error } objects for validation failures, throw for auth failures).
  </action>
