  <name>Task 1: Tenant CRUD server actions with cross-tenant access</name>
  <files>

  <action>
    Create `src/app/(admin)/actions/tenants.ts` with cross-tenant CRUD server actions:

    - Add 'use server' directive
    - Import `requireAuth`, `isSystemAdmin` from '@/lib/auth/server'
    - Import `prisma` from '@/lib/db/prisma' (base client, NOT getTenantPrisma -- critical for cross-tenant access)
    - Import `revalidatePath` from 'next/cache'
    - Import `z` from 'zod' for input validation

    **Helper function: requireSystemAdmin()**
    - Call `await requireAuth()` then `const admin = await isSystemAdmin()`
    - If not admin, throw `new Error('Unauthorized: System admin access required')`
    - Use this at the start of EVERY action (defense in depth -- layout check is UX-only per Phase 02 decision)

    **getAllTenants():**
    - Call requireSystemAdmin()
    - Use base `prisma.tenant.findMany()` with select: id, name, slug, isActive, createdAt, updatedAt
    - Include `_count: { select: { users: true, trucks: true, routes: true } }` for tenant health metrics
    - Order by createdAt desc
    - Return the result array

    **createTenant(formData: FormData):**
    - Call requireSystemAdmin()
    - Extract name and slug from FormData
    - Validate with Zod: name is required string (2-100 chars), slug is required string (2-50 chars, lowercase alphanumeric + hyphens only, regex: /^[a-z0-9]+(?:-[a-z0-9]+)*$/)
    - Use prisma.tenant.create with name, slug, isActive: true
    - revalidatePath('/tenants')
    - Return { success: true, tenant } on success, { success: false, error: string } on validation/duplicate failure
    - Catch Prisma unique constraint error (slug) and return friendly message

    **suspendTenant(tenantId: string):**
    - Call requireSystemAdmin()
    - Validate tenantId is non-empty string
    - Use prisma.tenant.update where id = tenantId, set isActive: false
    - revalidatePath('/tenants')
    - Return { success: true }

    **reactivateTenant(tenantId: string):**
    - Call requireSystemAdmin()
    - Validate tenantId is non-empty string
    - Use prisma.tenant.update where id = tenantId, set isActive: true
    - revalidatePath('/tenants')
    - Return { success: true }

    **deleteTenant(tenantId: string):**
    - Call requireSystemAdmin()
    - Validate tenantId is non-empty string
    - Use prisma.tenant.delete where id = tenantId (hard delete per project decision)
    - revalidatePath('/tenants')
    - Return { success: true }
    - Catch foreign key constraint errors and return friendly message about needing to remove users/trucks/routes first

    NOTE: All actions use the base `prisma` client (from '@/lib/db/prisma'), NOT getTenantPrisma(). This is critical because getTenantPrisma() applies RLS tenant scoping, but system admin needs to see ALL tenants.
  </action>
