1. Add Truck model to prisma/schema.prisma after the User model:

```prisma
model Truck {
  id                String   @id @default(dbgenerated("gen_random_uuid()")) @db.Uuid
  tenantId          String   @db.Uuid
  make              String
  model             String
  year              Int
  vin               String
  licensePlate      String
  odometer          Int      // Whole miles only (federal requirement)
  documentMetadata  Json?    // JSONB: registration number, insurance expiry, etc.
  createdAt         DateTime @default(now()) @db.Timestamptz
  updatedAt         DateTime @updatedAt @db.Timestamptz

  tenant            Tenant   @relation(fields: [tenantId], references: [id])

  @@unique([tenantId, vin])
  @@index([tenantId])
}
```

2. Add `trucks Truck[]` relation to the Tenant model (alongside existing `users User[]`).

3. Create migration SQL file manually (same pattern as init migration). The migration SQL must include:
   - CREATE TABLE "Truck" with all columns matching the Prisma model
   - Foreign key to Tenant
   - Indexes (tenantId, composite unique on tenantId+vin)
   - Enable RLS on Truck table (ENABLE ROW LEVEL SECURITY + FORCE ROW LEVEL SECURITY)
   - tenant_isolation_policy: USING ("tenantId" = current_tenant_id()) WITH CHECK ("tenantId" = current_tenant_id())
   - bypass_rls_policy: USING (current_setting('app.bypass_rls', TRUE)::text = 'on')
   - Follow exact same pattern as the User table RLS in the init migration

4. Run `npx prisma generate` to regenerate the Prisma client with the new Truck model.

IMPORTANT: Do NOT run `npx prisma migrate` -- there is no database running. Create the migration directory and SQL file manually, then generate the client.
