  <action>
    **1. Prisma schema changes (`prisma/schema.prisma`):**

    Add `DocumentType` enum before the Document model:
    ```
    enum DocumentType {
      DRIVER_LICENSE
      DRIVER_APPLICATION
      GENERAL
    }
    ```
    Note: Do NOT add TRUCK_REGISTRATION or TRUCK_INSURANCE — existing truck documents use JSONB metadata, not this enum. Keep it driver-focused to avoid migration complexity.

    Extend the `Document` model with these new fields (all optional for backwards compatibility with existing truck/route documents):
    - `driverId String? @db.Uuid` — optional driver ownership
    - `documentType DocumentType?` — categorize driver documents
    - `expiryDate DateTime? @db.Timestamptz` — track compliance expiry
    - `notes String?` — additional context (e.g., "State of CA")

    Add relation to User model:
    - On Document: `driver User? @relation(name: "DriverDocuments", fields: [driverId], references: [id])`
    - On User model: add `driverDocuments Document[] @relation(name: "DriverDocuments")` line

    Add indexes on Document model:
    - `@@index([driverId])`
    - `@@index([expiryDate])`

    Run `npx prisma db push` to apply schema changes (no migration needed for dev).

    **2. Update storage layer:**

    In `src/lib/storage/validate.ts`: Change `MAX_FILE_SIZE` from `10 * 1024 * 1024` to `100 * 1024 * 1024` (100MB). This is needed to support large scanned compliance PDFs.

    In `src/lib/storage/presigned.ts`: Add `'drivers'` to the `DocumentCategory` type so it reads `export type DocumentCategory = 'trucks' | 'routes' | 'drivers';`

    **3. Create multipart upload orchestration (`src/lib/storage/multipart.ts`):**

    Import from `@aws-sdk/client-s3`: `CreateMultipartUploadCommand`, `UploadPartCommand`, `CompleteMultipartUploadCommand`, `AbortMultipartUploadCommand`.
    Import from `@aws-sdk/s3-request-presigner`: `getSignedUrl`.
    Import `s3Client` and `bucket` from `./s3-client`.

    Install `@aws-sdk/lib-storage` first: `npm install @aws-sdk/lib-storage` (although we implement the multipart manually using presigned URLs per part, this package may be needed as a fallback — research recommends it).

    Actually, per the research, we do NOT use `@aws-sdk/lib-storage` for client-side uploads. Instead, we implement server-side orchestration with presigned URLs per part. The client uploads each part directly to R2 using the presigned URL.

    Export these functions:

    `initiateMultipartUpload(tenantId, category: DocumentCategory, fileId, fileName, contentType, totalParts)` — Creates an S3 multipart upload via `CreateMultipartUploadCommand`, returns `{ uploadId, s3Key, totalParts }`. The s3Key follows the existing pattern: `tenant-${tenantId}/${category}/${fileId}-${fileName}`.

    `getPartUploadUrl(s3Key, uploadId, partNumber)` — Returns a presigned URL (10-min expiry) for uploading a specific part via `UploadPartCommand`.

    `completeMultipartUpload(s3Key, uploadId, parts: Array<{ PartNumber, ETag }>)` — Completes the upload via `CompleteMultipartUploadCommand`.

    `abortMultipartUpload(s3Key, uploadId)` — Aborts and cleans up orphaned parts via `AbortMultipartUploadCommand`.

    **4. Update validation schema (`src/lib/validations/document.schemas.ts`):**

    Add `documentTypeEnum = z.enum(['DRIVER_LICENSE', 'DRIVER_APPLICATION', 'GENERAL'])`.

    Update `documentCreateSchema` to:
    - Add `driverId: z.string().uuid().optional()`
    - Add `documentType: documentTypeEnum.optional()`
    - Add `expiryDate: z.coerce.date().optional()` (allows string input, coerces to Date)
    - Add `notes: z.string().max(500).optional()`
    - Update the entity refinement: instead of `Boolean(data.truckId) !== Boolean(data.routeId)`, check that exactly ONE of `[data.truckId, data.routeId, data.driverId]` is truthy.
    - Add a second refinement: if `driverId` is set, `documentType` is required.

    Export `documentTypeEnum` and the updated types.

    **5. Update DocumentRepository (`src/lib/db/repositories/document.repository.ts`):**

    Add to `DocumentCreateInput` interface: `driverId?: string`, `documentType?: string`, `expiryDate?: Date`, `notes?: string`.

    Add `findByDriverId(driverId: string)` method that returns documents where `driverId` matches, ordered by `createdAt desc`, including `uploader` select (matching existing pattern). Also include `documentType`, `expiryDate`, `notes` in the select/return.

    Add `update(id: string, data: { expiryDate?: Date, notes?: string, documentType?: string })` method for editing document metadata.

    **1. Create multipart upload API routes:**

    These are API routes (not server actions) because the client needs to call them from XMLHttpRequest during chunked upload. All three routes require authentication and enforce tenant isolation.

    **`src/app/api/documents/multipart/initiate/route.ts`** — POST handler:
    - Authenticate user via `requireRole([UserRole.OWNER, UserRole.MANAGER])`.
    - Get `tenantId` from `requireTenantId()`.
    - Parse JSON body: `{ fileName, contentType, fileSize, entityType: 'driver', entityId, totalParts }`.
    - Validate: fileSize <= 100MB, contentType is in ALLOWED_TYPES, entityType is 'driver'.
    - Validate first bytes if provided (optional — client may not send magic bytes for multipart initiation; validation happens at complete step).
    - Generate fileId with `nanoid()`.
    - Sanitize fileName (remove path separators).
    - Call `initiateMultipartUpload(tenantId, 'drivers', fileId, sanitizedFileName, contentType, totalParts)`.
    - Return JSON: `{ uploadId, s3Key, fileId }`.

    **`src/app/api/documents/multipart/part-url/route.ts`** — POST handler:
    - Authenticate user via `requireRole([UserRole.OWNER, UserRole.MANAGER])`.
    - Get `tenantId` from `requireTenantId()`.
    - Parse JSON body: `{ s3Key, uploadId, partNumber }`.
    - **Defense-in-depth**: Verify `s3Key.startsWith(`tenant-${tenantId}/drivers/`)` — enforce tenant prefix AND drivers category.
    - Call `getPartUploadUrl(s3Key, uploadId, partNumber)`.
    - Return JSON: `{ uploadUrl }`.

    **`src/app/api/documents/multipart/complete/route.ts`** — POST handler:
    - Authenticate user via `requireRole([UserRole.OWNER, UserRole.MANAGER])`.
    - Get `tenantId` from `requireTenantId()` and `user` from `getCurrentUser()`.
    - Parse JSON body: `{ s3Key, uploadId, parts, fileName, contentType, sizeBytes, entityType: 'driver', entityId, documentType, expiryDate?, notes? }`.
    - **Defense-in-depth**: Verify `s3Key.startsWith(`tenant-${tenantId}/drivers/`)`.
    - Call `completeMultipartUpload(s3Key, uploadId, parts)`.
    - Validate document data with `documentCreateSchema`.
    - Create document record via `DocumentRepository.create()` with all fields including `driverId: entityId`, `documentType`, `expiryDate`, `notes`.
    - Revalidate `/drivers/${entityId}`.
    - Return JSON: `{ success: true, document }`.
    - If any step fails, attempt `abortMultipartUpload(s3Key, uploadId)` in catch block to clean up orphaned parts.

    **2. Create driver document server actions (`src/app/(owner)/actions/driver-documents.ts`):**

    `'use server'` directive at top.

    **`requestDriverUploadUrl(formData: FormData)`** — For small files (<5MB, using existing single-PUT pattern):
    - Mirror existing `requestUploadUrl` from `documents.ts` but:
      - Accept `entityType: 'driver'` and `driverId` from formData.
      - Accept `documentType` (DRIVER_LICENSE, DRIVER_APPLICATION, GENERAL) from formData.
      - Accept optional `expiryDate` and `notes` from formData.
      - Use category `'drivers'` for s3Key generation.
      - Return the same shape plus `documentType`, `expiryDate`, `notes` for the complete step.

    **`completeDriverDocumentUpload(data)`** — Save metadata after S3 upload:
    - Accept: `{ s3Key, fileName, contentType, sizeBytes, driverId, documentType, expiryDate?, notes? }`.
    - Auth check + tenant check.
    - **Defense-in-depth**: Verify `s3Key.startsWith(`tenant-${tenantId}/drivers/`)`.
    - Validate with `documentCreateSchema`.
    - Create via `DocumentRepository.create()`.
    - Revalidate `/drivers/${driverId}`.

    **`listDriverDocuments(driverId: string)`** — List documents for a driver:
    - Auth: OWNER, MANAGER, or DRIVER.
    - Return `DocumentRepository.findByDriverId(driverId)`.

    **`deleteDriverDocument(documentId: string)`** — Delete a driver document:
    - Auth: OWNER or MANAGER.
    - Find document, verify s3Key starts with `tenant-${tenantId}/drivers/` (defense-in-depth + entity ownership).
    - Delete from S3, then from database.
    - Revalidate driver page using the document's driverId.

    **`updateDriverDocument(documentId: string, data: { expiryDate?, notes?, documentType? })`** — Edit metadata:
    - Auth: OWNER or MANAGER.
    - Find document, verify ownership.
    - Update via `DocumentRepository.update()`.
    - Revalidate driver page.
