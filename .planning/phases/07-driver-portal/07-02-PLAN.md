**1. Create `src/components/driver/route-detail-readonly.tsx`** — Read-only route + truck display

This is a SERVER component (no 'use client'). It accepts a route object (with truck included) and formatted dates as props. Uses semantic HTML `dl/dt/dd` for data display (NOT disabled form inputs).

Structure:
- Section 1: "Route Details" card — origin, destination, scheduled date, status badge, completedAt (if present), notes (if present). Use the same status badge color logic as `RouteDetail` (PLANNED=gray, IN_PROGRESS=blue, COMPLETED=green). NO status transition buttons, NO edit link.
- Section 2: "Assigned Truck" card — vehicle (year make model), license plate, VIN, odometer (formatted with commas). NO "View Truck Details" link (driver shouldn't browse trucks).

Props interface:
```typescript
interface RouteDetailReadOnlyProps {
  route: {
    id: string;
    origin: string;
    destination: string;
    scheduledDate: Date;
    completedAt: Date | null;
    status: string;
    notes: string | null;
    driver: { id: string; firstName: string | null; lastName: string | null; email: string; licenseNumber: string | null; };
    truck: { id: string; make: string; model: string; year: number; vin: string; licensePlate: string; odometer: number; documentMetadata: any; };
  };
  formattedScheduledDate: string;
  formattedCompletedAt?: string;
}
```

Style with same card pattern as owner portal: `rounded-lg border border-gray-200 bg-white p-6 shadow-sm`. Use `text-sm font-medium text-gray-500` for labels, `text-sm text-gray-900` for values. Grid layout: `grid gap-4 sm:grid-cols-2`.

**2. Create `src/components/driver/document-list-readonly.tsx`** — Download-only document list

This is a CLIENT component (`'use client'`). It renders a list of documents with download buttons only (NO delete buttons, NO upload component).

Props:
```typescript
interface DocumentListReadOnlyProps {
  documents: Array<{
    id: string;
    fileName: string;
    contentType: string;
    sizeBytes: number;
    createdAt: Date;
  }>;
  downloadAction: (documentId: string) => Promise<{ downloadUrl: string; fileName: string } | { error: string }>;
}
```

Accept `downloadAction` as a prop so the component is reusable. The parent page will pass `getDriverDownloadUrl` as the action.

Reuse the same visual patterns from `DocumentList` (file type badges, file size formatting, date formatting) but:
- Only show Download button (blue), NO Delete button (red)
- No upload functionality
- Empty state: "No documents available." with document icon (same SVG as owner version)

Handle download: call `downloadAction(docId)`, check for error, open `downloadUrl` in new tab via `window.open`. Show "Downloading..." state while in progress.
