<objective>
Create driver-scoped server actions that enforce user-level data isolation on top of existing tenant-level RLS.

Purpose: The existing route and document server actions in (owner) allow DRIVER role but don't filter by the authenticated driver's ID — they return all tenant data. The driver portal needs dedicated server actions that resolve the driver's identity from `getCurrentUser()` and filter all queries by `driverId = user.id`. This is the critical IDOR-prevention layer.

Output: Two server action files providing driver-scoped data access for routes and documents.
</objective>

Create `src/app/(driver)/actions/driver-routes.ts` with a single server action `getMyAssignedRoute()`.

This action must:
1. Mark file with `'use server'` directive
2. Call `requireRole([UserRole.DRIVER])` FIRST before any data access
3. Call `getCurrentUser()` from `@/lib/auth/server` to get the database user record
4. If user is null, throw `new Error('User not found')`
5. Call `getTenantPrisma()` from `@/lib/context/tenant-context` to get tenant-scoped Prisma client
6. Query `prisma.route.findFirst()` with:
   - `where: { driverId: user.id, status: { in: ['PLANNED', 'IN_PROGRESS'] } }`
   - `include: { truck: { select: { id, make, model, year, vin, licensePlate, odometer, documentMetadata } }, driver: { select: { id, firstName, lastName, email, licenseNumber } } }`
   - `orderBy: { scheduledDate: 'asc' }` (earliest active route first)
7. Also include `documents: true` in the include to get route documents inline
8. Return the route (or null if no active assignment)

CRITICAL SECURITY RULES:
- NEVER accept routeId or driverId as a parameter — resolve identity from getCurrentUser()
- NEVER query all routes and filter client-side — filter in database WHERE clause
- The `driverId` in the WHERE clause MUST be `user.id` from the database user record, not from URL/params/formData

Follow the exact pattern from the research example (Pattern 1: Driver-Scoped Server Actions).

  <action>
Create `src/app/(driver)/actions/driver-documents.ts` with three server actions:

**1. `getMyRouteDocuments()`** — Returns documents for the driver's assigned route
- `requireRole([UserRole.DRIVER])` first
- `getCurrentUser()` to get user record
- `getTenantPrisma()` to get tenant-scoped client
- Query route first: `prisma.route.findFirst({ where: { driverId: user.id, status: { in: ['PLANNED', 'IN_PROGRESS'] } } })` to get the route ID
- If no route found, return empty array `[]`
- Use `DocumentRepository(tenantId).findByRouteId(route.id)` to fetch documents
- This double-check ensures driver can only access documents for their own route

**2. `getMyTruckDocuments()`** — Returns documents for the driver's assigned truck
- `requireRole([UserRole.DRIVER])` first
- `getCurrentUser()` to get user record
- `getTenantPrisma()` to get tenant-scoped client
- Query route first to get the truckId: `prisma.route.findFirst({ where: { driverId: user.id, status: { in: ['PLANNED', 'IN_PROGRESS'] } }, select: { truckId: true } })`
- If no route found, return empty array `[]`
- Use `DocumentRepository(tenantId).findByTruckId(route.truckId)` to fetch truck documents
- This ensures driver can only access documents for the truck assigned to their active route

**3. `getDriverDownloadUrl(documentId: string)`** — Generate presigned download URL with ownership verification
- `requireRole([UserRole.DRIVER])` first
- `getCurrentUser()` to get user record
- `requireTenantId()` to get tenant ID
- `DocumentRepository(tenantId).findById(documentId)` to fetch the document
- If document not found, return `{ error: 'Document not found' }`
- **CRITICAL ownership check:** Verify the document belongs to the driver's assigned route or truck:
  - `getTenantPrisma()` then query `prisma.route.findFirst({ where: { driverId: user.id, status: { in: ['PLANNED', 'IN_PROGRESS'] } }, select: { id: true, truckId: true } })`
  - Check: `doc.routeId === route.id || doc.truckId === route.truckId`
  - If neither matches, return `{ error: 'Document not accessible' }`
- Verify s3Key starts with `tenant-${tenantId}/` (defense in depth, same as owner action)
- Call `generateDownloadUrl(doc.s3Key)` from `@/lib/storage/presigned`
- Return `{ downloadUrl, fileName: doc.fileName }`

Import `requireTenantId` from `@/lib/context/tenant-context` for tenant ID access.
Import `generateDownloadUrl` from `@/lib/storage/presigned` for presigned URLs.
