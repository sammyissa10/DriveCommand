1. Run the production build and check bundle sizes:

2. Check the build output carefully:
- Look at the "First Load JS shared by all" value (framework + shared chunks)
- Look at each route's individual First Load JS value
- Record ALL values for the summary

3. If any route exceeds 500KB first-load JS, apply these optimizations in priority order:

**Priority 1: Verify dynamic imports are in place**
Check that Leaflet is behind dynamic import boundary:
```bash
grep -r "from 'leaflet'" src/ --include="*.ts" --include="*.tsx" | grep -v node_modules
grep -r "from 'react-leaflet'" src/ --include="*.ts" --include="*.tsx" | grep -v node_modules
```
Leaflet + react-leaflet should ONLY appear in src/components/maps/ files, which are behind `dynamic(() => import(...), { ssr: false })`. If any other file imports them directly, wrap in dynamic import.

**Priority 2: Check for @turf/turf in app code**
```bash
grep -r "from '@turf" src/ --include="*.ts" --include="*.tsx" | grep -v node_modules
```
Turf.js (~500KB) must only be in prisma/seeds/ (seed scripts), NOT in src/ app code. If found in app code, extract the specific function needed and use a lighter alternative.

**Priority 3: Dynamic import chart components if needed**
If safety or fuel routes exceed budget, dynamically import the chart components:
```tsx
import dynamic from 'next/dynamic';
const SafetyScoreCard = dynamic(() => import('@/components/safety/safety-score-card').then(m => ({ default: m.SafetyScoreCard })));
```
Apply to each chart component import on the offending page. This defers Recharts from first-load JS to on-demand loading.

**Priority 4: Install @next/bundle-analyzer for detailed analysis (only if needed)**
```bash
npm install --save-dev @next/bundle-analyzer
```
Then wrap next.config.ts:
```typescript
import type { NextConfig } from 'next';
import bundleAnalyzer from '@next/bundle-analyzer';

const withBundleAnalyzer = bundleAnalyzer({
  enabled: process.env.ANALYZE === 'true',
});

const nextConfig: NextConfig = {
  experimental: {
    serverActions: {
      bodySizeLimit: '10mb',
    },
  },
};

export default withBundleAnalyzer(nextConfig);
```
Then run `ANALYZE=true npx next build` for detailed treemap visualization.

4. After any optimizations, re-run `npx next build` and verify ALL routes show First Load JS < 500KB.

5. If bundle is already under 500KB without changes (likely since charts are client components behind page boundaries and Leaflet is already dynamically imported), document this finding.

6. Record final bundle sizes in the summary.
