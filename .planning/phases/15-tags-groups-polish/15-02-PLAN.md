2. Update server actions to accept optional tagId parameter for filtering:

In src/app/(owner)/live-map/actions.ts:
- `getLatestVehicleLocations(tagId?: string)`: If tagId provided, add INNER JOIN to TagAssignment table:
  ```sql
  INNER JOIN "TagAssignment" ta ON ta."truckId" = t.id AND ta."tagId" = ${tagId}::uuid
  ```
  This filters to only trucks with that tag assignment. If no tagId, query remains unchanged.

In src/app/(owner)/safety/actions.ts:
- Add optional `tagId?: string` parameter to `getFleetSafetyScore`, `getEventDistribution`, `getSafetyScoreTrend`, `getDriverRankings`
- When tagId provided, add a subquery filter: `AND se."truckId" IN (SELECT "truckId" FROM "TagAssignment" WHERE "tagId" = ${tagId}::uuid AND "truckId" IS NOT NULL)`
- For getDriverRankings, filter the Truck JOIN with the same subquery

In src/app/(owner)/fuel/actions.ts:
- Add optional `tagId?: string` parameter to all 5 server actions
- When tagId provided, add same subquery filter pattern: `AND fr."truckId" IN (SELECT "truckId" FROM "TagAssignment" WHERE "tagId" = ${tagId}::uuid AND "truckId" IS NOT NULL)`
- For getCO2Emissions and getIdleTimeAnalysis (which use LEFT JOIN on Truck), add `AND t.id IN (SELECT "truckId" FROM "TagAssignment" WHERE ...)` to the WHERE clause
