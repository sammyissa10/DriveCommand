1. Add Tag and TagAssignment models to prisma/schema.prisma:

```prisma
model Tag {
  id        String   @id @default(dbgenerated("gen_random_uuid()")) @db.Uuid
  tenantId  String   @db.Uuid
  name      String
  color     String   @default("#3b82f6") // Tailwind blue-500
  createdAt DateTime @default(now()) @db.Timestamptz
  updatedAt DateTime @updatedAt @db.Timestamptz

  tenant      Tenant          @relation(fields: [tenantId], references: [id])
  assignments TagAssignment[]

  @@unique([tenantId, name])
  @@index([tenantId])
}

model TagAssignment {
  id        String   @id @default(dbgenerated("gen_random_uuid()")) @db.Uuid
  tenantId  String   @db.Uuid
  tagId     String   @db.Uuid
  truckId   String?  @db.Uuid
  userId    String?  @db.Uuid
  createdAt DateTime @default(now()) @db.Timestamptz

  tenant Tenant @relation(fields: [tenantId], references: [id])
  tag    Tag    @relation(fields: [tagId], references: [id], onDelete: Cascade)
  truck  Truck? @relation(fields: [truckId], references: [id], onDelete: Cascade)
  user   User?  @relation(fields: [userId], references: [id], onDelete: Cascade)

  @@unique([tagId, truckId])
  @@unique([tagId, userId])
  @@index([tenantId])
  @@index([tagId])
  @@index([truckId])
  @@index([userId])
}
```

Add relation arrays to existing models:
- Tenant: add `tags Tag[]` and `tagAssignments TagAssignment[]`
- Truck: add `tagAssignments TagAssignment[]`
- User: add `tagAssignments TagAssignment[]`

2. Create migration SQL at prisma/migrations/20260215000002_add_tags/migration.sql following the EXACT pattern from prisma/migrations/20260215000001_add_fleet_intelligence_models/migration.sql:
- CREATE TABLE "Tag" with all columns, UUID default gen_random_uuid(), UNIQUE on ("tenantId", "name")
- CREATE TABLE "TagAssignment" with all columns, UUID default gen_random_uuid(), UNIQUE on ("tagId", "truckId"), UNIQUE on ("tagId", "userId")
- All indexes matching the @@index definitions
- Foreign keys: "tenantId" -> "Tenant"(id), "tagId" -> "Tag"(id) ON DELETE CASCADE, "truckId" -> "Truck"(id) ON DELETE CASCADE, "userId" -> "User"(id) ON DELETE CASCADE
- RLS policies following existing pattern (copy from any existing migration):
  ```sql
  ALTER TABLE "Tag" ENABLE ROW LEVEL SECURITY;
  CREATE POLICY "tenant_isolation_policy" ON "Tag" USING ("tenantId"::text = current_setting('app.current_tenant_id', TRUE));
  CREATE POLICY "bypass_rls_policy" ON "Tag" USING (current_setting('app.bypass_rls', TRUE) = 'on');
  ```
  Same two policies for "TagAssignment".
- Add CHECK constraint to ensure exactly one entity per assignment:
  ```sql
  ALTER TABLE "TagAssignment" ADD CONSTRAINT "one_entity_per_assignment"
    CHECK ((("truckId" IS NOT NULL)::int + ("userId" IS NOT NULL)::int) = 1);
  ```

4. Create src/lib/validations/tag.schemas.ts with Zod schemas (import z from 'zod'):
- `createTagSchema`: z.object({ name: z.string().trim().min(1).max(50), color: z.string().regex(/^#[0-9a-fA-F]{6}$/) })
- `assignTagSchema`: z.object({ tagId: z.string().uuid(), truckId: z.string().uuid().optional(), userId: z.string().uuid().optional() }).refine(data => (data.truckId && !data.userId) || (!data.truckId && data.userId), { message: 'Exactly one of truckId or userId must be provided' })
- Export a `PRESET_COLORS` array of 8 hex colors for the UI color picker: ['#3b82f6' (blue), '#22c55e' (green), '#ef4444' (red), '#eab308' (yellow), '#a855f7' (purple), '#f97316' (orange), '#ec4899' (pink), '#14b8a6' (teal)]

5. Create src/app/(owner)/actions/tags.ts with 'use server' directive. Follow the EXACT pattern from src/app/(owner)/actions/trucks.ts for auth/tenant setup:
- Import { requireRole } from '@/lib/auth/server', { UserRole } from '@/lib/auth/roles', { getTenantPrisma, requireTenantId } from '@/lib/context/tenant-context'
- `createTag(formData: FormData)`: requireRole OWNER/MANAGER, extract name+color from formData, validate with createTagSchema, get tenantId via requireTenantId(), insert via db.tag.create({ data: { tenantId, name, color } }), revalidatePath('/tags')
- `deleteTag(tagId: string)`: requireRole OWNER/MANAGER, db.tag.delete({ where: { id: tagId } }) (cascade deletes assignments), revalidatePath('/tags')
- `listTags()`: requireRole OWNER/MANAGER, db.tag.findMany({ orderBy: { name: 'asc' } }) - RLS auto-scopes to tenant
- `listTagsWithAssignments()`: requireRole OWNER/MANAGER, db.tag.findMany({ include: { assignments: { include: { truck: { select: { id: true, make: true, model: true, licensePlate: true } }, user: { select: { id: true, firstName: true, lastName: true, email: true } } } } }, orderBy: { name: 'asc' } })
- `assignTag(formData: FormData)`: requireRole OWNER/MANAGER, extract tagId+truckId+userId from formData, validate with assignTagSchema, get tenantId, db.tagAssignment.create({ data: { tenantId, tagId, truckId: truckId || null, userId: userId || null } }), revalidatePath('/tags')
- `unassignTag(assignmentId: string)`: requireRole OWNER/MANAGER, db.tagAssignment.delete({ where: { id: assignmentId } }), revalidatePath('/tags')
- Use @ts-ignore before each Prisma call (same pattern as existing actions for Prisma 7 type issues)
