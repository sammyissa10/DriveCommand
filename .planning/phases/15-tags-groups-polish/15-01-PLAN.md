1. Add Tag and TagAssignment models to prisma/schema.prisma:

```prisma
model Tag {
  id        String   @id @default(dbgenerated("gen_random_uuid()")) @db.Uuid
  tenantId  String   @db.Uuid
  name      String
  color     String   @default("#3b82f6") // Tailwind blue-500
  createdAt DateTime @default(now()) @db.Timestamptz
  updatedAt DateTime @updatedAt @db.Timestamptz

  tenant      Tenant          @relation(fields: [tenantId], references: [id])
  assignments TagAssignment[]

  @@unique([tenantId, name])
  @@index([tenantId])
}

model TagAssignment {
  id        String   @id @default(dbgenerated("gen_random_uuid()")) @db.Uuid
  tenantId  String   @db.Uuid
  tagId     String   @db.Uuid
  truckId   String?  @db.Uuid
  userId    String?  @db.Uuid
  createdAt DateTime @default(now()) @db.Timestamptz

  tenant Tenant @relation(fields: [tenantId], references: [id])
  tag    Tag    @relation(fields: [tagId], references: [id], onDelete: Cascade)
  truck  Truck? @relation(fields: [truckId], references: [id], onDelete: Cascade)
  user   User?  @relation(fields: [userId], references: [id], onDelete: Cascade)

  @@unique([tagId, truckId])
  @@unique([tagId, userId])
  @@index([tenantId])
  @@index([tagId])
  @@index([truckId])
  @@index([userId])
}
```

Add relation arrays to Tenant (`tags Tag[]`, `tagAssignments TagAssignment[]`), Truck (`tagAssignments TagAssignment[]`), and User (`tagAssignments TagAssignment[]`).

2. Create migration SQL at prisma/migrations/20260215000002_add_tags/migration.sql:
- CREATE TABLE "Tag" with all columns, UUID PK, UNIQUE on (tenantId, name)
- CREATE TABLE "TagAssignment" with all columns, UUID PK, UNIQUE on (tagId, truckId), UNIQUE on (tagId, userId)
- All indexes matching the @@index definitions
- RLS policies following existing pattern:
  ```sql
  ALTER TABLE "Tag" ENABLE ROW LEVEL SECURITY;
  CREATE POLICY "tenant_isolation_policy" ON "Tag" USING ("tenantId"::text = current_setting('app.current_tenant_id', TRUE));
  CREATE POLICY "bypass_rls_policy" ON "Tag" USING (current_setting('app.bypass_rls', TRUE) = 'on');
  ```
  Same pattern for TagAssignment.
- Foreign keys to Tenant, Tag, Truck, User with ON DELETE CASCADE for tag and entity FKs.

3. Run `npx prisma generate` to update the Prisma client.

4. Create src/lib/validations/tag.schemas.ts with Zod schemas:
- `createTagSchema`: name (string, 1-50 chars, trimmed), color (string, hex color pattern /^#[0-9a-fA-F]{6}$/)
- `assignTagSchema`: tagId (uuid), truckId (optional uuid), userId (optional uuid) with .refine() ensuring exactly one of truckId or userId is provided

5. Create src/app/(owner)/actions/tags.ts with 'use server':
- `createTag(formData)`: requireRole OWNER/MANAGER, validate with createTagSchema, insert via getTenantPrisma, revalidatePath('/tags')
- `deleteTag(tagId)`: requireRole OWNER/MANAGER, delete tag (cascade deletes assignments), revalidatePath('/tags')
- `listTags()`: requireRole OWNER/MANAGER, fetch all tags for tenant ordered by name
- `listTagsWithAssignments()`: requireRole OWNER/MANAGER, fetch tags with include: { assignments: { include: { truck: { select: { id, make, model, licensePlate } }, user: { select: { id, firstName, lastName, email } } } } }
- `assignTag(formData)`: requireRole OWNER/MANAGER, validate with assignTagSchema, create TagAssignment, revalidatePath('/tags')
- `unassignTag(assignmentId)`: requireRole OWNER/MANAGER, delete TagAssignment by id, revalidatePath('/tags')
- `getTagsForEntity(entityType: 'truck' | 'user', entityId: string)`: requireRole OWNER/MANAGER, fetch tags assigned to specific truck or user
- Follow exact pattern from safety/actions.ts: getTenantPrisma(), @ts-ignore for Prisma 7 type issues

Use predefined color palette: provide a PRESET_COLORS array of 8 colors (blue, green, red, yellow, purple, orange, pink, teal as hex values) for the UI to pick from.
