<task type="auto">
  <name>Task 2: Create onboarding page and UserMenu component for sign-out</name>
  <files>
    src/app/onboarding/page.tsx

  <action>
1. Create `src/app/onboarding/page.tsx`:
   - This is a Server Component that handles the race condition between signup and webhook processing.
   - Import `auth` from `@clerk/nextjs/server` and `redirect` from `next/navigation`.
   - Call `auth()` to get `userId` and `sessionClaims`.
   - If no userId, redirect to `/sign-in`.
   - Check `sessionClaims.privateMetadata.tenantId` — if it exists, redirect to `/dashboard` (tenant is provisioned, onboarding complete).
   - If tenantId is NOT set yet, render a waiting page with:
     - "Setting up your workspace..." heading
     - A brief message: "This usually takes a few seconds."
     - A client component or meta refresh that refreshes the page every 3 seconds to check if the webhook has completed. Use a simple approach: `<meta httpEquiv="refresh" content="3" />` in the page. This is the simplest approach that avoids client-side JavaScript complexity.
   - The middleware already redirects authenticated users without tenantId to /onboarding, so this page is the landing spot.

2. Create `src/components/navigation/user-menu.tsx`:
   - A client component (`'use client'`).
   - Import `UserButton` from `@clerk/nextjs`.
   - Render `<UserButton afterSignOutUrl="/" />`.
   - Style the avatar box to be reasonably sized: `appearance={{ elements: { avatarBox: 'h-9 w-9' } }}`.
   - Export as `UserMenu` component.
   - This component will be used in portal layouts (Plan 02-03) and anywhere sign-out is needed.

3. Update `src/app/page.tsx`:
   - Replace the minimal placeholder with a basic landing page that provides navigation to sign-in and sign-up.
   - Import `auth` from `@clerk/nextjs/server` and `redirect` from `next/navigation`.
   - If user is authenticated (has userId), redirect to `/dashboard`.
   - If not authenticated, show a simple landing page with:
     - "DriveCommand" heading
     - "Logistics fleet management platform" tagline
     - Two links/buttons: "Sign In" (href="/sign-in") and "Sign Up" (href="/sign-up")
   - Use Tailwind for basic styling. Keep it minimal — this is not a marketing page.

Why onboarding uses meta refresh: The webhook race condition means the first page load after signup may not have tenantId yet. A meta refresh is the simplest server-side approach — no client JS, no WebSocket, no polling API. The page re-renders server-side every 3 seconds, re-checks sessionClaims, and redirects when ready.
