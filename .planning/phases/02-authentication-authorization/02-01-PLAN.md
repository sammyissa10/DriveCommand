  <name>Task 1: Create role definitions and server-side authorization helpers</name>
  <files>

    package.json
  </files>
  <action>
1. Install Zod: `npm install zod` (needed for server action input validation in this phase and all future phases).

2. Create `src/lib/auth/roles.ts`:
   - Export a `UserRole` enum matching the Prisma UserRole enum values: OWNER, MANAGER, DRIVER, SYSTEM_ADMIN. Note: Prisma has OWNER, MANAGER, DRIVER. SYSTEM_ADMIN is NOT in Prisma enum because system admins are identified by `isSystemAdmin` boolean. Include SYSTEM_ADMIN in the TypeScript enum for authorization logic but do NOT add it to Prisma UserRole enum.
   - Export a `ROLE_HIERARCHY` constant mapping each role to a numeric level for comparison (SYSTEM_ADMIN: 100, OWNER: 50, MANAGER: 40, DRIVER: 10).
   - Export `PORTAL_ROLES` constant grouping which roles can access which portal: admin portal requires SYSTEM_ADMIN, owner portal allows OWNER and MANAGER, driver portal allows DRIVER.

3. Create `src/lib/auth/server.ts`:
   - `getRole()`: async function that calls `auth()` from `@clerk/nextjs/server`, reads `sessionClaims.publicMetadata.role` as UserRole. Returns null if no session or no role. If user's database `isSystemAdmin` flag needs checking, that's a separate concern (handled by `isSystemAdmin()` function).
   - `requireAuth()`: async function that calls `auth()`, throws `Error('Unauthorized: Authentication required')` if no userId. Returns userId string.
   - `requireRole(allowedRoles: UserRole[])`: async function that calls `getRole()`, throws `Error('Unauthorized: Required roles: ...')` if role not in allowedRoles. Returns the matched role.
   - `isSystemAdmin()`: async function that calls `auth()` to get userId, then queries the database User record by clerkUserId to check the `isSystemAdmin` boolean. Uses the base `prisma` client (no RLS) since system admins may not have a tenant context. Returns boolean.
   - `getCurrentUser()`: async function that calls `auth()` to get userId, returns null if not authenticated. Calls `prisma.user.findUnique({ where: { clerkUserId } })` using base prisma (no RLS). Returns the User record or null.
   - All functions use `import { auth } from '@clerk/nextjs/server'`.

Important: `getRole()` reads from Clerk publicMetadata (fast, no DB call). `isSystemAdmin()` reads from database (requires DB call, used sparingly). This dual approach balances speed with the isSystemAdmin field that lives in the database only.
