**Prisma Schema Changes (prisma/schema.prisma):**

1. Add `InvitationStatus` enum with values: `PENDING`, `ACCEPTED`, `EXPIRED`, `CANCELLED`

2. Extend the `User` model with new nullable fields (nullable because OWNER/MANAGER roles don't need them):
   - `firstName String?` — driver's first name
   - `lastName String?` — driver's last name
   - `licenseNumber String?` — driver's license number (permissive format)
   - `isActive Boolean @default(true)` — for deactivation without deletion
   - Add `@@index([email])` for invitation lookup by email

3. Add `DriverInvitation` model:
   - `id String @id @default(dbgenerated("gen_random_uuid()")) @db.Uuid`
   - `tenantId String @db.Uuid`
   - `email String`
   - `firstName String`
   - `lastName String`
   - `licenseNumber String?` — optional, permissive validation
   - `clerkInvitationId String @unique` — Clerk invitation ID for tracking
   - `expiresAt DateTime @db.Timestamptz` — Clerk default 30 days
   - `status InvitationStatus @default(PENDING)`
   - `acceptedAt DateTime? @db.Timestamptz` — set when driver accepts
   - `userId String? @db.Uuid` — links to User record after acceptance
   - `createdAt DateTime @default(now()) @db.Timestamptz`
   - `updatedAt DateTime @updatedAt @db.Timestamptz`
   - Relation: `tenant Tenant @relation(fields: [tenantId], references: [id])`
   - Indexes: `@@index([tenantId])`, `@@index([email])`, `@@index([clerkInvitationId])`

4. Add `driverInvitations DriverInvitation[]` relation to `Tenant` model

**Zod Validation Schemas (src/lib/validations/driver.schemas.ts):**

1. `driverInviteSchema`:
   - `email`: z.string().email('Invalid email address')
   - `firstName`: z.string().min(1, 'First name is required').max(50)
   - `lastName`: z.string().min(1, 'Last name is required').max(50)
   - `licenseNumber`: z.string().min(5, 'License number too short').max(20, 'License number too long').regex(/^[A-Z0-9\s\-]+$/i, 'License number contains invalid characters').optional().or(z.literal('')) — permissive validation per research, empty string treated as undefined

2. `driverUpdateSchema`:
   - `firstName`: z.string().min(1, 'First name is required').max(50)
   - `lastName`: z.string().min(1, 'Last name is required').max(50)
   - `licenseNumber`: z.string().min(5, 'License number too short').max(20, 'License number too long').regex(/^[A-Z0-9\s\-]+$/i, 'License number contains invalid characters').optional().or(z.literal(''))

3. Export types: `DriverInvite`, `DriverUpdate` (inferred from schemas)

**Server Actions (src/app/(owner)/actions/drivers.ts):**

File must start with `'use server';` directive. Follow the exact same pattern as `src/app/(owner)/actions/trucks.ts` — requireRole as FIRST operation, then validation, then data access.

1. `inviteDriver(prevState: any, formData: FormData)`:
   - `await requireRole([UserRole.OWNER, UserRole.MANAGER])` as FIRST line
   - Parse FormData: email, firstName, lastName, licenseNumber
   - Validate with `driverInviteSchema.safeParse()`, return `{ error: fieldErrors }` on failure
   - Get tenantId via `requireTenantId()` and prisma via `getTenantPrisma()`
   - Check for existing user with same email in tenant: `prisma.user.findFirst({ where: { email, tenantId } })` — if exists, return `{ error: { email: ['A user with this email already exists in your organization'] } }`
   - Cancel any pending invitations for same email: `prisma.driverInvitation.updateMany({ where: { email, tenantId, status: 'PENDING' }, data: { status: 'CANCELLED' } })`
   - Create Clerk invitation: `const clerk = await clerkClient(); clerk.invitations.createInvitation({ emailAddress, redirectUrl: process.env.NEXT_PUBLIC_APP_URL + '/sign-up', publicMetadata: { role: UserRole.DRIVER, firstName, lastName }, privateMetadata: { tenantId } })`
   - Store DriverInvitation record: `prisma.driverInvitation.create({ data: { tenantId, email, firstName, lastName, licenseNumber: result.data.licenseNumber || null, clerkInvitationId: invitation.id, expiresAt: new Date(Date.now() + 30 * 24 * 60 * 60 * 1000), status: 'PENDING' } })`
   - `revalidatePath('/drivers')`
   - Return `{ success: true, message: 'Invitation sent to ' + email }`
   - Wrap Clerk API call in try/catch — on error return `{ error: 'Failed to send invitation. Please try again.' }`

2. `listDrivers()`:
   - `await requireRole([UserRole.OWNER, UserRole.MANAGER])`
   - Get prisma via `getTenantPrisma()`
   - Return `prisma.user.findMany({ where: { role: 'DRIVER' }, orderBy: { createdAt: 'desc' } })`

3. `getDriver(id: string)`:
   - `await requireRole([UserRole.OWNER, UserRole.MANAGER])`
   - Get prisma via `getTenantPrisma()`
   - Return `prisma.user.findUnique({ where: { id } })` — RLS handles tenant scoping

4. `updateDriver(id: string, prevState: any, formData: FormData)`:
   - `await requireRole([UserRole.OWNER, UserRole.MANAGER])`
   - Parse FormData: firstName, lastName, licenseNumber
   - Validate with `driverUpdateSchema.safeParse()`, return errors on failure
   - Clean licenseNumber: if empty string, set to null
   - Get prisma via `getTenantPrisma()`
   - `prisma.user.update({ where: { id }, data: { firstName, lastName, licenseNumber } })`
   - `revalidatePath('/drivers')` and `revalidatePath('/drivers/' + id)`
   - Return `{ success: true }`

5. `deactivateDriver(id: string)`:
   - `await requireRole([UserRole.OWNER, UserRole.MANAGER])`
   - Get prisma via `getTenantPrisma()`
   - Update user: `prisma.user.update({ where: { id }, data: { isActive: false } })`
   - Revoke all Clerk sessions: `const clerk = await clerkClient(); const sessions = await clerk.sessions.getSessionList({ userId: user.clerkUserId }); for (const session of sessions.data) { await clerk.sessions.revokeSession(session.id); }`
   - Note: Use `user.clerkUserId` from the updated user record to find sessions. The Clerk API `getSessionList` takes a `userId` filter parameter (this is the Clerk user ID, not our database user ID).
   - `revalidatePath('/drivers')`
   - Return `{ success: true }`

6. `reactivateDriver(id: string)`:
   - `await requireRole([UserRole.OWNER, UserRole.MANAGER])`
   - Get prisma via `getTenantPrisma()`
   - `prisma.user.update({ where: { id }, data: { isActive: true } })`
   - `revalidatePath('/drivers')`
   - Return `{ success: true }`

7. `listInvitations()`:
   - `await requireRole([UserRole.OWNER, UserRole.MANAGER])`
   - Get prisma via `getTenantPrisma()`
   - Return `prisma.driverInvitation.findMany({ orderBy: { createdAt: 'desc' } })` — RLS handles tenant scoping

Import from: `@clerk/nextjs/server` for clerkClient, `@/lib/auth/server` for requireRole, `@/lib/auth/roles` for UserRole, `@/lib/context/tenant-context` for getTenantPrisma/requireTenantId, `@/lib/validations/driver.schemas` for schemas, `next/cache` for revalidatePath.
