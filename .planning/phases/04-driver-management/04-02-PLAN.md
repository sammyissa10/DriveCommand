Extend the existing webhook handler at `src/app/api/webhooks/clerk/route.ts` to handle driver sign-up from invitations. The current webhook only handles owner provisioning (creates Tenant + User with OWNER role). Add a driver path that:

**Inside the existing `if (evt.type === 'user.created')` block, AFTER extracting clerkUserId and email, BEFORE the existing tenant provisioning logic:**

1. Extract metadata from event data:
   ```
   const { private_metadata } = evt.data;
   const inviteRole = evt.data.public_metadata?.role as string | undefined;
   const inviteTenantId = private_metadata?.tenantId as string | undefined;
   ```

2. Check if this is a driver invitation sign-up (has DRIVER role AND tenantId in metadata):
   ```
   if (inviteRole === 'DRIVER' && inviteTenantId) {
     // Driver invitation flow - handle separately
   }
   ```

3. **Driver invitation flow** (inside the if block):
   - Use `prisma` (NOT tenant-scoped — webhook runs without tenant context) to find pending invitation:
     ```
     const invitation = await prisma.driverInvitation.findFirst({
       where: { email, tenantId: inviteTenantId, status: 'PENDING' },
     });
     ```
   - If no invitation found, log error and return 400: `{ error: 'Driver accounts require an invitation' }`
   - Check invitation expiry: if `new Date() > invitation.expiresAt`, update invitation status to 'EXPIRED' and return 400: `{ error: 'Invitation has expired' }`
   - Create User record with DRIVER role:
     ```
     const user = await prisma.user.create({
       data: {
         clerkUserId,
         email,
         tenantId: inviteTenantId,
         role: 'DRIVER',
         firstName: invitation.firstName,
         lastName: invitation.lastName,
         licenseNumber: invitation.licenseNumber,
         isActive: true,
       },
     });
     ```
   - Mark invitation as accepted:
     ```
     await prisma.driverInvitation.update({
       where: { id: invitation.id },
       data: { status: 'ACCEPTED', acceptedAt: new Date(), userId: user.id },
     });
     ```
   - Update Clerk metadata for the new user (publicMetadata.role = DRIVER, privateMetadata.tenantId):
     ```
     const client = await clerkClient();
     await client.users.updateUserMetadata(clerkUserId, {
       publicMetadata: { role: 'DRIVER' },
       privateMetadata: { tenantId: inviteTenantId },
     });
     ```
   - Return 200: `{ message: 'Driver account created', userId: user.id }`

4. **Idempotency:** The existing `existingUser` check at the top of the handler already handles duplicate webhooks. If user already exists, it updates metadata and returns 200. This covers the driver case too.

5. **Important:** The driver flow must come BEFORE the existing tenant-creation flow. If the sign-up is a driver invitation (has role=DRIVER and tenantId), skip the tenant creation entirely. Use an early return inside the driver block.

6. Wrap the driver creation in try/catch, return 500 on error with message "Failed to create driver account".

Do NOT modify the existing owner provisioning logic below the driver block. The existing code should only execute if the sign-up is NOT a driver invitation (i.e., it's a new owner sign-up).
