Create Zod validation schemas following the pattern in route.schemas.ts:

```typescript
import { z } from 'zod';

export const expenseCreateSchema = z.object({
  routeId: z.string().uuid('Invalid route ID'),
  categoryId: z.string().uuid('Invalid category'),
  amount: z.coerce
    .number()
    .positive('Amount must be positive')
    .finite('Amount must be finite')
    .multipleOf(0.01, 'Amount must have at most 2 decimal places')
    .max(999999.99, 'Amount cannot exceed $999,999.99'),
  description: z.string().min(1, 'Description is required').max(200),
  notes: z.string().max(1000).optional(),
});

export const expenseUpdateSchema = expenseCreateSchema.omit({ routeId: true }).partial();

export type ExpenseCreate = z.infer<typeof expenseCreateSchema>;
export type ExpenseUpdate = z.infer<typeof expenseUpdateSchema>;
```

Create server actions following the exact pattern from routes.ts (requireRole first, FormData parsing, Zod validation, revalidatePath):

1. `createExpense(prevState: any, formData: FormData)` — OWNER/MANAGER only.
   - Parse routeId, categoryId, amount, description, notes from FormData.
   - Validate with expenseCreateSchema.
   - Check route exists AND route.status !== 'COMPLETED' (return error if completed: "Cannot add expenses to a completed route").
   - Create RouteExpense with `amount: new Decimal(result.data.amount)`.
   - Use `import { Decimal } from '@prisma/client/runtime/library'` for Decimal conversion.
   - Revalidate `/routes/${routeId}` and `/routes`.
   - Return `{ success: true }` or `{ error: ... }`.

2. `updateExpense(expenseId: string, prevState: any, formData: FormData)` — OWNER/MANAGER only.
   - Parse categoryId, amount, description, notes from FormData.
   - Validate with expenseUpdateSchema.
   - Fetch existing expense to get routeId. Check the expense's route is not COMPLETED.
   - Check `deletedAt` is null (can't edit soft-deleted expense).
   - Update expense. Convert amount to Decimal if provided.
   - Revalidate paths.
   - Return `{ success: true }` or `{ error: ... }`.

3. `deleteExpense(expenseId: string)` — OWNER/MANAGER only.
   - Soft delete: update deletedAt to `new Date()`. Do NOT use prisma.delete().
   - Fetch expense first to get routeId. Check route is not COMPLETED.
   - Revalidate paths.
   - Return `{ success: true }` or `{ error: ... }`.

4. `listExpenses(routeId: string)` — OWNER/MANAGER/DRIVER read access.
   - Find all expenses where `routeId` matches and `deletedAt: null`.
   - Include category relation (`select: { id: true, name: true }`).
   - Order by createdAt desc.
   - Return the list.

5. `listExpenseCategories()` — OWNER/MANAGER/DRIVER read access.
   - Return all ExpenseCategory records for the tenant.
   - Order by isSystemDefault desc (system defaults first), then name asc.

CRITICAL: Use `import { Decimal } from '@prisma/client/runtime/library'` — NOT `decimal.js` directly. This matches the Prisma Decimal integration pattern.
CRITICAL: All financial mutations must check route.status !== 'COMPLETED' before allowing changes.
