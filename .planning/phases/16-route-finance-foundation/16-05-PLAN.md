Add two new functions to the existing route-calculator.ts file (do not remove existing functions):

1. `calculateCostPerMile(totalExpenses: Decimal | string, startOdometer: number | null, endOdometer: number | null)`:
   - If startOdometer or endOdometer is null, return `{ costPerMile: null, miles: null }` (distance unknown).
   - Calculate miles = endOdometer - startOdometer.
   - If miles <= 0, return `{ costPerMile: null, miles: 0 }` (invalid data).
   - Calculate costPerMile = totalExpenses / miles using Decimal.div().toDecimalPlaces(2).
   - Return `{ costPerMile: string (Decimal as fixed 2), miles: number }`.

2. `compareToFleetAverage(routeCostPerMile: string | null, fleetAverage: string | null)`:
   - If either is null, return `{ comparison: 'unknown', difference: null, differencePercent: null }`.
   - Calculate difference = routeCostPerMile - fleetAverage.
   - Calculate differencePercent = ((routeCostPerMile - fleetAverage) / fleetAverage) * 100.
   - Handle zero-division for fleetAverage = 0.
   - Return `{ comparison: 'above' | 'below' | 'equal', difference: string, differencePercent: number }`.
   - Use Decimal arithmetic for all calculations.

All new functions MUST use `import { Decimal } from '@prisma/client/runtime/library'` for arithmetic.
