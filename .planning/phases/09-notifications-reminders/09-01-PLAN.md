    **5. Create the cron API route** at `src/app/api/cron/send-reminders/route.ts`:

    ```typescript
    export const dynamic = 'force-dynamic'; // CRITICAL: Prevent Next.js caching

    export async function GET(request: NextRequest) {
      // 1. Verify CRON_SECRET (Bearer token in Authorization header)
      const authHeader = request.headers.get('authorization');
      if (authHeader !== `Bearer ${process.env.CRON_SECRET}`) {
        return new Response('Unauthorized', { status: 401 });
      }

      // 2. Get all active tenants (bypass RLS - system operation)
      // Use raw prisma client, set bypass_rls in transaction
      const tenants = await prisma.$transaction(async (tx) => {
        await tx.$executeRaw`SELECT set_config('app.bypass_rls', 'on', TRUE)`;
        return tx.tenant.findMany({ where: { isActive: true }, select: { id: true, name: true } });
      });

      // 3. For each tenant:
      //    a. Find OWNER-role users (notification recipients)
      //    b. Find upcoming maintenance (7 days)
      //    c. Find expiring documents (14 days)
      //    d. For each item, generate idempotency key, check dedup, send email, log result

      // 4. Return JSON summary: { success: true, maintenance: { sent, skipped, failed }, documents: { sent, skipped, failed } }
    }
    ```

    The cron handler should:

    - Use `prisma.$extends(withTenantRLS(tenantId))` for tenant-scoped queries (trucks, scheduled services, users)
    - Find OWNER-role users per tenant: `tenantPrisma.user.findMany({ where: { role: 'OWNER' } })`
    - Send to ALL owners per tenant (research recommendation)
    - Continue processing next tenant if one tenant fails (try/catch per tenant)
