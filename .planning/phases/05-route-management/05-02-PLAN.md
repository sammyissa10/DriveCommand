- `submitLabel`: string (e.g., "Create Route" or "Update Route")

Form fields:

- **Destination**: text input, required, max 200 chars
- **Scheduled Date**: datetime-local input, required

- **Truck**: select dropdown populated from `trucks` prop. Display format: "Year Make Model (LicensePlate)". Empty option: "Select a truck..."
- **Notes**: textarea, optional, max 1000 chars

Field-level validation errors displayed from state.error (keyed by field name), matching the truck-form.tsx pattern. Disabled inputs while isPending. Button text shows "Saving..." during pending. Pre-fills all fields from initialData when provided.

**2. RouteList (src/components/routes/route-list.tsx):**

'use client' component using TanStack Table. Props:
- `routes`: array of route objects with included driver and truck data (matching listRoutes() return shape)
- `onDelete`: callback function `(id: string) => void`

TanStack Table setup with useReactTable, getCoreRowModel, getSortedRowModel, getFilteredRowModel.

State: sorting (SortingState), columnFilters (ColumnFiltersState), globalFilter (string).

Columns:
- **Origin**: accessorKey 'origin'
- **Destination**: accessorKey 'destination'
- **Scheduled Date**: accessorKey 'scheduledDate', cell formatter converts Date to readable string using `new Intl.DateTimeFormat('en-US', { month: 'short', day: 'numeric', year: 'numeric' }).format(new Date(value))`
- **Driver**: accessorFn combining `row.driver.firstName + ' ' + row.driver.lastName`
- **Truck**: accessorFn combining `row.truck.make + ' ' + row.truck.model + ' (' + row.truck.licensePlate + ')'`
- **Status**: accessorKey 'status', cell renders colored badge:
  - PLANNED: `bg-gray-100 text-gray-800`
  - IN_PROGRESS: `bg-blue-100 text-blue-800`
  - COMPLETED: `bg-green-100 text-green-800`
  - Display text: replace underscores with spaces
- **Actions**: view link to `/routes/{id}`, delete button (red text, window.confirm before calling onDelete)

Filter UI above table:
- Global search input: "Search routes..." placeholder
- Status filter dropdown: "All Statuses", "Planned", "In Progress", "Completed" - sets column filter on 'status' column

Table styling matches truck-list.tsx: full width, border-collapse, gray header bg, hover rows, clickable column headers with sort arrows (text arrows, not emoji).

Empty state: "No routes found. Create your first route to get started." centered in white card.

Server component (NO 'use client'). This page needs to fetch active drivers and trucks for the form dropdowns. Fetch active drivers: `prisma.user.findMany({ where: { role: 'DRIVER', isActive: true }, select: { id: true, firstName: true, lastName: true } })`. Fetch trucks: `prisma.truck.findMany({ select: { id: true, make: true, model: true, year: true, licensePlate: true } })`. Use getTenantPrisma() and requireRole() for auth. Page heading: "Create Route" with back link to /routes. Renders NewRouteClient with drivers and trucks arrays.

**4. NewRouteClient (src/app/(owner)/routes/new/new-route-client.tsx):**
'use client' component. Props: drivers, trucks. Renders RouteForm with action={createRoute}, drivers, trucks, submitLabel="Create Route". White card background with shadow and border.

**5. Route Detail Page (src/app/(owner)/routes/[id]/page.tsx):**
Server component. Extract id from params (async pattern from Phase 03). Call getRoute(id). If null, return notFound(). Format dates server-side using formatDateInTenantTimezone() and tenant timezone (hardcode 'UTC' for now since we don't have tenant timezone lookup in actions - the Tenant model has timezone field but accessing it requires separate query; use 'UTC' default for v1, can enhance later). Page header: "{origin} to {destination}" as title with "Edit Route" button linking to /routes/{id}/edit and "Back to Routes" link. Render RouteDetail component with route data, formatted dates. Also render RouteStatusActions with routeId, currentStatus, and updateRouteStatus action.

**6. Edit Route Page (src/app/(owner)/routes/[id]/edit/page.tsx):**
Server component. Extract id from params (async). Call getRoute(id). If null, notFound(). Also fetch active drivers and trucks (same queries as new page). Page heading: "Edit Route" with back link to /routes/{id}. Format scheduledDate using formatForDatetimeInput() for the datetime-local input pre-fill. Render EditRouteClient with routeId, initialData (origin, destination, scheduledDate formatted string, driverId, truckId, notes), drivers, trucks.

**7. EditRouteClient (src/app/(owner)/routes/[id]/edit/edit-route-client.tsx):**
'use client' component. Props: routeId, initialData, drivers, trucks. Binds updateRoute with routeId: `updateRoute.bind(null, routeId)`. Renders RouteForm with bound action, initialData, drivers, trucks, submitLabel="Update Route".

**Navigation Flow:**
- List -> New: "Create Route" button

- Detail -> Edit: "Edit Route" button
- Detail -> List: "Back to Routes" link
- Edit -> Detail: "Back to Route" link
- New -> List: "Back to Routes" link

**Server Actions Integration:**
- createRoute: Used in new page, redirects to /routes/{id} on success

- listRoutes: Fetches all routes in list page
- getRoute: Fetches single route in detail and edit pages

- Use `import { getTenantPrisma } from '@/lib/context/tenant-context'` for data fetching in server component pages
- For the new and edit pages that fetch drivers/trucks directly, call requireRole() first, then getTenantPrisma()
