  <action>
Create `src/app/(owner)/actions/settlements.ts` with `'use server'` directive. Import patterns from existing actions (invoices.ts, payroll.ts):
- `import { Prisma } from '@/generated/prisma';`
- `const Decimal = Prisma.Decimal;`
- `import { getTenantPrisma, requireTenantId } from '@/lib/context/tenant-context';`
- `import { TX_OPTIONS } from '@/lib/db/prisma';`
- `import { requireRole } from '@/lib/auth/server';`

**Actions to implement:**

1. `listDrivers()` — list active drivers in tenant (for dropdown). Return id, firstName, lastName.

2. `upsertDriverPayConfig(driverId: string, payType: string, rateValue: string, effectiveFrom: string)` — Create or update driver pay configuration. Validate payType is PER_MILE|PERCENTAGE|FLAT. Validate rateValue is a positive number. Create a new DriverPayConfig record (never update old ones — history is preserved via effectiveFrom). requireRole OWNER/MANAGER.

3. `getDriverPayConfig(driverId: string)` — Fetch the most recent (by effectiveFrom desc) pay config for a driver. Return null if none.

4. `calculateSettlement(formData: FormData)` — Main engine. Extract driverId, periodStart, periodEnd from FormData.
   - requireRole OWNER/MANAGER
   - Fetch most recent DriverPayConfig for driver where effectiveFrom <= periodEnd
   - If no config found, return `{ error: 'No pay configuration found for this driver.' }`
   - Fetch all loads where driverId matches, status is 'DELIVERED', and deliveryDate (or updatedAt if no deliveryDate — check Load model) is between periodStart and periodEnd
   - If no loads found, return `{ error: 'No delivered loads found in this period.' }`
   - For each load, calculate pay using Decimal:
     - PER_MILE: `new Decimal(payConfig.rateValue).mul(new Decimal(load.distanceMiles ?? 0))`
     - PERCENTAGE: `new Decimal(load.rate).mul(new Decimal(payConfig.rateValue)).div(100)`
     - FLAT: `new Decimal(payConfig.rateValue)`
   - Build settlement lines array with: loadId, description (`${load.loadNumber}: ${load.origin} -> ${load.destination}`), miles, grossRate, payAmount
   - Sum totalPay using Decimal.reduce
   - Create DriverSettlement + SettlementLines in `prisma.$transaction(..., TX_OPTIONS)`
   - Handle P2002 (unique constraint violation) with error: 'A settlement already exists for this driver and period.'
   - revalidatePath('/payroll/settlements')
   - Return `{ settlementId: created.id }`

5. `listSettlements()` — Fetch all settlements for tenant, include driver (firstName, lastName), order by createdAt desc. requireRole OWNER/MANAGER.

6. `getSettlement(id: string)` — Fetch single settlement with driver info and all lines (include load with loadNumber). requireRole OWNER/MANAGER.

7. `approveSettlement(id: string)` — Update status from DRAFT to APPROVED. Only allow if current status is DRAFT. requireRole OWNER.

8. `markSettlementPaid(id: string)` — Update status from APPROVED to PAID, set paidAt to now(). Only allow if current status is APPROVED. requireRole OWNER.

9. `deleteSettlement(id: string)` — Delete settlement. Only allow if status is DRAFT. requireRole OWNER. Cascade deletes lines automatically.

CRITICAL: All money math uses `Prisma.Decimal` — never parseFloat or JS number for calculations. The `Decimal` alias pattern (`const Decimal = Prisma.Decimal`) is standard in this codebase.
