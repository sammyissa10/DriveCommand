**Settlement PDF component** at `src/components/settlements/settlement-pdf.tsx`:
- Use .tsx extension (required for JSX syntax with react-pdf elements).
- NO `'use client'` directive — this is a server-only component imported by the Route Handler.
- Import `{ Document, Page, View, Text, StyleSheet }` from `@react-pdf/renderer`.
- Use `StyleSheet.create()` for all styles (react-pdf does not support Tailwind).
- Font: Helvetica (built-in, no download needed — matches existing rate confirmation PDF pattern from quick-25).

**PDF layout:**
- Page size: LETTER, padding 40
- Header section: "Driver Settlement Statement" title (fontSize 18, bold), driver name, pay period (formatted with date-fns `format(date, 'MMM dd, yyyy')`), status, settlement ID
- Table header row: gray background, columns: "Load #", "Route (Origin -> Dest)", "Miles", "Gross Rate", "Driver Pay"
- Line item rows: one per SettlementLine, alternating white/light-gray background for readability
- All monetary values formatted with `$` prefix and 2 decimal places (use `Number(x).toFixed(2)`)
- Miles formatted with comma separators
- Total row: bold, border-top-2 black, shows total pay amount
- Pay type note below total: "Pay Type: PER_MILE @ $X.XXXX/mi" or "PERCENTAGE @ X.XX%" or "FLAT @ $X.XX/load"
- Signature section at bottom (marginTop 60): signature line (horizontal rule, width 250), "Driver Signature" label, date line

**Props interface:**
```typescript
interface SettlementPDFProps {
  settlement: {
    id: string;
    periodStart: Date;
    periodEnd: Date;
    status: string;
    totalPay: any; // Prisma.Decimal
    driver: { firstName: string; lastName: string };
    lines: Array<{
      id: string;
      description: string;
      miles: any;
      grossRate: any;
      payAmount: any;
      load?: { loadNumber: string };
    }>;
  };
  payType?: string;
  rateValue?: string;
}
```

**Route Handler** at `src/app/api/settlements/[id]/pdf/route.ts`:
- Export `GET` function following the exact pattern from research:
  ```typescript
  export async function GET(
    req: NextRequest,
    { params }: { params: Promise<{ id: string }> }
  )
  ```
- Authenticate via `getSession()` — return 401 if no session.
- Fetch settlement from DB using `getTenantPrisma()` (ensures tenant isolation via RLS):
  - Include driver (firstName, lastName), lines (include load with loadNumber)
  - Also fetch the driver's pay config for the pay type note
- If settlement not found, return 404 JSON.
- Render PDF: `const buffer = await renderToBuffer(React.createElement(SettlementPDF, { settlement, payType, rateValue }));`
- Cast as `as any` if TypeScript complains about ReactElement<DocumentProps> generic constraint (same pattern used in quick-25 rate confirmation PDF).
- Return `new Response(buffer, { headers: { 'Content-Type': 'application/pdf', 'Content-Disposition': 'attachment; filename="settlement-${id}.pdf"' } })`.

**Wire download button on settlement detail page:**
- Update `src/app/(owner)/payroll/settlements/[id]/page.tsx` to ensure the "Download PDF" button/link points to `/api/settlements/${settlement.id}/pdf` with `download` attribute.
- Show the download button for ALL statuses (DRAFT, APPROVED, PAID) — useful for review before approval.

CRITICAL: Use `renderToBuffer` (not `renderToString` which is deprecated, not `renderToStream` which has type issues). Import React and use `React.createElement()` syntax in the route handler.
