Define the Prisma schema in `prisma/schema.prisma` with these models:

**Tenant model:**
- `id`: String, UUID, primary key, `@default(dbgenerated("gen_random_uuid()"))`, `@db.Uuid`
- `name`: String (company name, required)
- `slug`: String? (optional, unique — for future flexibility)
- `timezone`: String, `@default("UTC")` (tenant's display timezone)
- `isActive`: Boolean, `@default(true)` (for admin to suspend tenants)
- `createdAt`: DateTime, `@default(now())`, `@db.Timestamptz`
- `updatedAt`: DateTime, `@updatedAt`, `@db.Timestamptz`

**User model (tenant-scoped):**
- `id`: String, UUID, primary key, `@default(dbgenerated("gen_random_uuid()"))`, `@db.Uuid`
- `tenantId`: String, `@db.Uuid` (FK to Tenant)
- `clerkUserId`: String, unique (maps to Clerk user)
- `email`: String
- `role`: Enum `UserRole` { OWNER, MANAGER, DRIVER }
- `isSystemAdmin`: Boolean, `@default(false)` (platform-level admin flag — per discretion decision)
- `createdAt`: DateTime, `@default(now())`, `@db.Timestamptz`
- `updatedAt`: DateTime, `@updatedAt`, `@db.Timestamptz`
- Relation: `tenant Tenant @relation(fields: [tenantId], references: [id])`
- Indexes: `@@index([tenantId])`, `@@index([clerkUserId])`

**Discretion decisions applied here:**
- UUID primary keys (not slugs) for security — slug is optional field on Tenant
- Hard deletes (no `deletedAt` field) — simpler for v1
- No audit logging tables — deferred to Phase 2+
- Foundation tables only (Tenant, User) — entity tables (Truck, Driver, Route) added in their phases
- `isSystemAdmin` boolean on User — platform-level admin, not special tenant
- Tenant has `timezone` field per locked user decision
- All timestamps use `@db.Timestamptz` — store UTC, display in tenant timezone

Create the migration using `npx prisma migrate dev --name init --create-only` to generate the SQL migration without applying it.

Then manually append RLS policies to the generated migration SQL file. Add the following SQL AFTER the Prisma-generated table creation:

```sql
-- ==============================================
-- Row-Level Security Setup
-- ==============================================

-- 1. Helper function to get current tenant from session variable
CREATE OR REPLACE FUNCTION current_tenant_id()
RETURNS UUID AS $$
  SELECT NULLIF(current_setting('app.current_tenant_id', TRUE), '')::UUID;
$$ LANGUAGE SQL STABLE;

-- 2. Enable RLS on User table (tenant-scoped)
ALTER TABLE "User" ENABLE ROW LEVEL SECURITY;
ALTER TABLE "User" FORCE ROW LEVEL SECURITY;

-- 3. Tenant isolation policy for User
CREATE POLICY tenant_isolation_policy ON "User"
  FOR ALL
  USING (tenant_id = current_tenant_id())
  WITH CHECK (tenant_id = current_tenant_id());

-- 4. Bypass policy for system admin / provisioning operations
CREATE POLICY bypass_rls_policy ON "User"
  FOR ALL
  USING (current_setting('app.bypass_rls', TRUE)::text = 'on');

-- Note: Tenant table does NOT have RLS — it's the root entity.
-- Tenants are accessed by ID from Clerk metadata, not queried broadly.
-- System admins query all tenants; owners only access their own via ID lookup.
```

IMPORTANT: Use `tenant_id` as the column name in the RLS policy SQL (matching the database column name that Prisma generates from `tenantId`). Prisma's default mapping converts camelCase to snake_case in the database. If using `@map("tenant_id")` on the field, use that. Otherwise check the generated SQL to confirm the actual column name and match accordingly.

Do NOT apply the migration yet — it requires a running PostgreSQL instance which the user will configure.
