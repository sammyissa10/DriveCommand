    **File 1: `prisma/seeds/gps-locations.ts`**

    Export `seedGPSLocations(tenantId: string, trucks: { id: string }[])`.

    For each truck:
    1. Pick 3-5 random route pairs from ROUTE_PAIRS
    2. For each route, generate a GPS trail:
       - Random start time within the last 30 days
       - Use `@turf/turf` to create a LineString between origin and destination coordinates
       - Use `turf.length(line, { units: 'miles' })` to calculate total distance
       - Calculate realistic trip duration (distance / 60 mph average)
       - Use `turf.along(line, distance, { units: 'miles' })` to interpolate points every 1 mile
       - For each point: compute timestamp (proportional to distance), speed (55-65 mph with variation), heading (bearing to next point via `turf.bearing()`), altitude (100-2000m random), accuracy (5-15m random)
       - Store as Prisma.Decimal for lat/lng (use `.toFixed(8)`)
    3. Batch all GPS records for the tenant
    4. Insert using `createMany()` within an RLS transaction:
       ```typescript
       await prisma.$transaction(async (tx) => {
         await tx.$executeRaw`SELECT set_config('app.current_tenant_id', ${tenantId}, TRUE)`;
         await tx.gPSLocation.createMany({ data: allLocations });
       });
       ```

    IMPORTANT: Turf.js uses [lng, lat] coordinate order (GeoJSON standard), NOT [lat, lng]. Be careful with the coordinate ordering when creating LineStrings and extracting results.

    IMPORTANT: The GPS points per route could be very large (1000+ miles = 1000+ points). For 25 trucks x 4 routes x 500 avg points = 50K records. This is fine for createMany but ensure you're not hitting memory limits. If total exceeds 50K, reduce to 1 point every 2 miles.

    Add jitter to GPS trails for realism: small random offsets (0.001-0.005 degrees, ~100-500m) to simulate GPS drift and non-straight-line travel. Apply this via faker random offsets to the interpolated coordinates.

    **File 2: `prisma/seeds/safety-events.ts`**

    Export `seedSafetyEvents(tenantId: string, trucks: { id: string }[], gpsLocations: { truckId: string; latitude: any; longitude: any; timestamp: Date }[])`.

    For each truck:
    1. Get this truck's GPS locations
    2. Generate 5-15 random safety events per truck
    3. For each event, pick a random GPS location from the truck's trail (so the event occurs ON the route, not at a random location)
    4. Pick random event type: weight toward HARSH_BRAKING (30%), SPEEDING (30%), HARSH_ACCELERATION (20%), HARSH_CORNERING (10%), others (10%)
    5. Severity: SPEEDING tends HIGH/CRITICAL, HARSH_BRAKING tends MEDIUM/HIGH, others tend LOW/MEDIUM
    6. G-force: only for HARSH_BRAKING/ACCELERATION/CORNERING events (0.5-2.5g range)
    7. Speed: 45-95 mph (higher for speeding events)
    8. SpeedLimit: only for SPEEDING events (55-75 mph, lower than actual speed)
    9. Metadata JSON: context field (Highway, City streets, Residential, Interstate)
    10. driverId and routeId: null (we don't have route assignment data in seeds — these will be populated in production)

    Insert via createMany in RLS transaction (same pattern).

    **File 3: `prisma/seeds/fuel-records.ts`**

    Export `seedFuelRecords(tenantId: string, trucks: { id: string; odometer: number }[])`.

    For each truck:
    1. Generate 8-12 fuel fill-up records over last 30 days
    2. Start from truck's current odometer
    3. Each fill-up adds 200-400 miles to odometer
    4. Fuel type: DIESEL for all (fleet trucks)
    5. Quantity: 50-150 gallons per fill-up
    6. Unit cost: $3.20-$4.50/gallon (realistic diesel prices)
    7. Total cost: quantity * unitCost
    8. Location: random from realistic truck stop names (Love's Travel Stop, Pilot Flying J, TA Petro, Shell, Chevron)
    9. Timestamp: distributed evenly over last 30 days (fill-up every 2-4 days)
    10. isEstimated: false for all (actual fill-ups)
    11. Latitude/longitude: null (fuel station coordinates not critical for Phase 11)

    Insert via createMany in RLS transaction.

    IMPORTANT: Use `Prisma.Decimal` for all Decimal fields (quantity, unitCost, totalCost). Use `new Prisma.Decimal(value.toFixed(N))` pattern.

    IMPORTANT: The `odometer` field on the truck in the Prisma schema is an Int (whole miles). Make sure fuel record odometer values are integers.
