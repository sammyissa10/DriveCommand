    2. **Create tables:** GPSLocation, SafetyEvent, FuelRecord with:
       - UUID primary keys with `gen_random_uuid()`
       - All column types matching Prisma schema exactly
       - Foreign key constraints to Tenant (ON DELETE RESTRICT), Truck (ON DELETE RESTRICT), User (ON DELETE SET NULL for optional driverId), Route (ON DELETE SET NULL for optional routeId)
       - DEFAULT values (now() for createdAt, false for isEstimated)

    3. **Create indexes:** All indexes matching the `@@index` declarations in schema:
       - Single column indexes on tenantId, truckId, timestamp for all three tables
       - Composite `[tenantId, timestamp]` indexes for all three tables
       - Additional indexes: driverId, routeId, eventType, severity on SafetyEvent; isEstimated on FuelRecord

    4. **Enable RLS** (CRITICAL — same pattern as all v1.0 tables):
       ```sql
       -- For each of the three tables:
       ALTER TABLE "TableName" ENABLE ROW LEVEL SECURITY;
       ALTER TABLE "TableName" FORCE ROW LEVEL SECURITY;

       CREATE POLICY tenant_isolation_policy ON "TableName"
         FOR ALL
         USING ("tenantId" = current_tenant_id())
         WITH CHECK ("tenantId" = current_tenant_id());

       CREATE POLICY bypass_rls_policy ON "TableName"
         FOR ALL
         USING (current_setting('app.bypass_rls', TRUE)::text = 'on');
       ```

    5. **Add foreign keys** using ALTER TABLE (matching Prisma's migration pattern):
       - GPSLocation: tenantId -> Tenant(id), truckId -> Truck(id)
       - SafetyEvent: tenantId -> Tenant(id), truckId -> Truck(id), driverId -> User(id) ON DELETE SET NULL, routeId -> Route(id) ON DELETE SET NULL
       - FuelRecord: tenantId -> Tenant(id), truckId -> Truck(id)

    IMPORTANT: The `current_tenant_id()` function already exists from the init migration. Do NOT recreate it.

    IMPORTANT: Use RESTRICT (not CASCADE) for required foreign keys (tenantId, truckId) to match v1.0 pattern. Use SET NULL for optional foreign keys (driverId, routeId).
