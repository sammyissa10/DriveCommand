**Create `src/components/maintenance/maintenance-event-form.tsx`** — 'use client' component:
- Follow the exact form pattern from `src/components/trucks/truck-form.tsx`
- Props: `action: (prevState: any, formData: FormData) => Promise<any>`, `currentOdometer: number`, `submitLabel: string`
- Use `useActionState(action, null)` for form state management
- Fields:
  - serviceType: text input, required, placeholder "e.g., Oil Change, Tire Rotation"
  - serviceDate: date input, defaultValue = today (new Date().toISOString().split('T')[0]), max = today (prevent future dates for historical events), required
  - odometerAtService: number input, defaultValue = currentOdometer, min=0, step=1, required. Show helper text "Current odometer: X miles"
  - cost: number input, min=0, step=0.01, placeholder "0.00", NOT required
  - provider: text input, placeholder "e.g., Joe's Auto Shop", NOT required
  - notes: textarea, rows=3, NOT required
- Submit button with isPending state ("Saving..." when pending)
- Display field-level validation errors from state.error using the same pattern as TruckForm

**Create `src/components/maintenance/scheduled-service-list.tsx`** — 'use client' component:
- Follow TanStack Table pattern from truck-list.tsx
- Props: `schedules` array (each with id, serviceType, intervalDays, intervalMiles, baselineDate, baselineOdometer, dueStatus from calculateNextDue), `onDelete: (id: string) => void`
- Columns:
  - Service Type (string)
  - Interval: display human-readable text like "Every 90 days" and/or "Every 5,000 mi" (handle both, one, or the other)
  - Next Due Date: show dueStatus.nextDueDate formatted, or '—' if no time trigger
  - Days Until Due: show dueStatus.daysUntilDue number (color red if <= 0, yellow if <= 14)
  - Next Due Mileage: show dueStatus.nextDueMileage toLocaleString + " mi", or '—' if no mileage trigger
  - Miles Until Due: show dueStatus.milesUntilDue toLocaleString (color red if <= 0, yellow if <= 500)
  - Status: badge showing "Due" (red) if isDue, "Upcoming" (yellow) if within threshold, "OK" (green) otherwise
- Delete button per row with confirm dialog
- Default sorting: by "status" putting due items first, or by nextDueDate ascending
- Global filter input

  - intervalDays: number input, min=1, step=1, placeholder "e.g., 90", NOT required. Label "Time Interval (days)"
  - intervalMiles: number input, min=1, step=1, placeholder "e.g., 5000", NOT required. Label "Mileage Interval (miles)"
  - Show helper text: "At least one interval is required. Service is due when EITHER trigger is met."
  - baselineDate: date input, defaultValue = today, required. Label "Baseline Date (last service or today)"
  - baselineOdometer: number input, defaultValue = currentOdometer, min=0, step=1, required. Label "Baseline Odometer"
  - notes: textarea, rows=3, NOT required
