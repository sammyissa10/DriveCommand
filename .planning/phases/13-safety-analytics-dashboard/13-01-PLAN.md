    **Step 1: Create `src/lib/safety/score-calculator.ts`** with these exports:

    - `SEVERITY_WEIGHTS`: Record mapping SafetyEventSeverity to penalty points (LOW=1, MEDIUM=2, HIGH=4, CRITICAL=8). Import SafetyEventSeverity from `@/generated/prisma`.
    - `calculateSafetyScore(eventCounts: Record<SafetyEventSeverity, number>): number`: Returns `Math.max(0, Math.round(100 - totalPenalty))` where totalPenalty = sum of count * weight for each severity.
    - `getScoreColor(score: number): string`: Returns Tailwind text color class: >=80 green-600, >=60 yellow-600, >=40 orange-600, else red-600.
    - `getScoreLabel(score: number): string`: Returns label string: >=80 "Excellent", >=60 "Good", >=40 "Needs Improvement", else "Poor".

    Follow the exact implementation from the research document (Pattern 4).

    **Step 2: Create `src/app/(owner)/safety/actions.ts`** with `'use server'` directive and these 4 server actions. Each action starts with `await requireRole([UserRole.OWNER, UserRole.MANAGER])` and uses `getTenantPrisma()` + `requireTenantId()` for RLS-scoped queries.

    Follow the established pattern from `src/app/(owner)/live-map/actions.ts`: use `@ts-ignore` before raw queries, cast results as `any[]`, convert Decimal fields to Number().

    **Action 1: `getFleetSafetyScore(daysBack: number = 30)`**
    - Raw SQL: `SELECT severity, COUNT(*)::int as count FROM "SafetyEvent" WHERE "tenantId" = ${tenantId}::uuid AND timestamp >= ${cutoff} GROUP BY severity`
    - Build counts object `{ LOW: 0, MEDIUM: 0, HIGH: 0, CRITICAL: 0 }` from results
    - Call `calculateSafetyScore(counts)` for the score
    - Return `{ score, totalEvents, eventsBySeverity: counts, period: daysBack }`

    **Action 2: `getEventDistribution(daysBack: number = 30)`**
    - Raw SQL: `SELECT "eventType", COUNT(*)::int as count FROM "SafetyEvent" WHERE "tenantId" = ${tenantId}::uuid AND timestamp >= ${cutoff} GROUP BY "eventType" ORDER BY count DESC`
    - Map enum values to human-readable labels using EVENT_LABELS constant (HARSH_BRAKING -> "Harsh Braking", etc. for all 8 types)
    - Also query severity distribution: `SELECT severity, COUNT(*)::int as count FROM "SafetyEvent" WHERE ... GROUP BY severity`
    - Return `{ byType: [{eventType, label, count}], bySeverity: [{severity, label, count, fill}] }` where fill is the CSS variable for chart colors (e.g., `var(--color-low)`)

    **Action 3: `getSafetyScoreTrend(daysBack: number = 30)`**
    - Raw SQL: `SELECT DATE(timestamp) as date, severity, COUNT(*)::int as count FROM "SafetyEvent" WHERE ... GROUP BY DATE(timestamp), severity ORDER BY date ASC`
    - Initialize all dates in the range with zero counts (so days with no events show score=100)
    - For each day, calculate score using `calculateSafetyScore()` on that day's event counts
    - Return array of `{ date: string (YYYY-MM-DD), score: number, events: number }`

    **Action 4: `getDriverRankings(daysBack: number = 30)`**
    - Aggregate by TRUCK (since seed data has driverId=null). Join SafetyEvent with Truck table.
    - Raw SQL with LEFT JOIN to include trucks with zero events:
      ```sql
      SELECT t.id as "truckId", t.make, t.model, t."licensePlate",
        COALESCE(SUM(CASE WHEN se.severity = 'LOW' THEN 1 ELSE 0 END), 0)::int as low_count,
        COALESCE(SUM(CASE WHEN se.severity = 'MEDIUM' THEN 1 ELSE 0 END), 0)::int as medium_count,
        COALESCE(SUM(CASE WHEN se.severity = 'HIGH' THEN 1 ELSE 0 END), 0)::int as high_count,
        COALESCE(SUM(CASE WHEN se.severity = 'CRITICAL' THEN 1 ELSE 0 END), 0)::int as critical_count,
        COUNT(se.id)::int as total_events
      FROM "Truck" t
      LEFT JOIN "SafetyEvent" se ON t.id = se."truckId" AND se.timestamp >= ${cutoff} AND se."tenantId" = ${tenantId}::uuid
      WHERE t."tenantId" = ${tenantId}::uuid
      GROUP BY t.id, t.make, t.model, t."licensePlate"
      ORDER BY total_events ASC
      ```
    - Calculate score per truck using `calculateSafetyScore()` with the 4 severity counts
    - Return array of `{ truckId, name (make + model), licensePlate, score, totalEvents }` sorted by score descending (best first)
